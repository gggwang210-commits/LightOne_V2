# 트레이너·센터 고객 인터뷰 질문지

이 문서는 LIGHTONE V2를 **비의료 PT 상담 보고서 SaaS**로 검증하기 위한 고객 인터뷰 스크립트입니다. 인터뷰 대상은 PT샵 대표, 헬스장 대표, 필라테스 센터장, 트레이너입니다. 파일럿 운영 설계는 [파일럿 검증 계획](pilot-validation-plan.md), 개인정보 기준은 [개인정보 보호 체크리스트](../governance/privacy-checklist.md)를 함께 확인합니다.

## 1. 인터뷰 목적

다음 핵심 가설이 실제 현장 문제와 구매 의사로 이어지는지 확인합니다.

1. 트레이너는 상담 전 자료 정리에 시간을 쓰고 있다.
2. 재등록 상담 때 회원 변화를 설명할 구조화된 자료가 부족하다.
3. 센터장은 트레이너별 상담 품질 표준화에 관심이 있다.
4. 월 구독 또는 리포트당 과금에 대한 지불의사가 있는지 검증해야 한다.
5. AI 단독 판단보다 트레이너 검토 기반 보고서가 더 수용 가능하다.

## 2. 진행 원칙

- 제품 판매보다 문제 검증을 우선합니다.
- 유도 질문을 피하고 실제 최근 사례를 묻습니다.
- 실제 회원 이름, 연락처, 사진, 상세 식별 정보는 수집하지 않습니다.
- 필요한 경우 “A회원”, “B센터” 같은 가상 표현만 사용합니다.
- 통증 관련 내용은 운동 중 반응 기록과 상담 참고 정보로만 다룹니다.
- AI는 초안 생성과 정리 보조 역할이며, 최종 확인은 트레이너가 수행한다고 설명합니다.

## 3. 인터뷰 대상별 확인 포인트

| 대상 | 핵심 확인 포인트 |
|------|------------------|
| PT샵 대표 | 재등록 상담 구조, 팀 내 상담 품질 편차, 비용 지불 기준 |
| 헬스장 대표 | 여러 트레이너의 기록 표준화, 운영 관리, 도입 장벽 |
| 필라테스 센터장 | 회원 변화 설명 방식, 수업 기록 관리, 상담 리포트 필요성 |
| 트레이너 | 상담 준비 시간, 기록 분산 문제, 보고서 사용 가능성 |

## 4. 오프닝 멘트

안녕하세요. LIGHTONE V2는 PT샵·헬스장·필라테스 센터가 회원의 운동 기록, 생활습관, 수업 메모를 정리해 상담 보고서로 활용하도록 돕는 비의료 PT 상담 보고서 SaaS입니다. 오늘 인터뷰는 판매가 아니라 현장 문제와 파일럿 가능성을 확인하기 위한 자리입니다. 실제 개인정보는 수집하지 않고, 답변은 비식별화해 정리하겠습니다.

## 5. 배경 질문

1. 현재 역할은 무엇인가요? 예: 센터 대표, 센터장, 트레이너
2. 주로 담당하는 회원 유형은 어떻게 되나요?
3. 회원 상담은 보통 언제 진행하나요? 예: 최초 등록, 중간 점검, 재등록 전
4. 상담 자료는 현재 어디에 기록하나요? 예: 메모 앱, 카카오톡, 종이, 센터 관리 시스템
5. 상담 자료를 다시 찾거나 정리하는 데 불편했던 최근 사례가 있나요?

## 6. 문제 검증 질문

### 가설 1. 상담 전 자료 정리 시간

- 상담 전 회원 기록을 확인하는 데 보통 얼마나 시간이 걸리나요? `[확인필요]`
- 가장 오래 걸리는 작업은 무엇인가요?
- 자료가 흩어져 있어 상담 준비가 늦어진 최근 사례가 있나요?
- 이 시간이 줄어든다면 어떤 업무에 시간을 더 쓰고 싶나요?

### 가설 2. 재등록 상담 자료 부족

- 재등록 상담에서 회원 변화는 어떻게 설명하나요?
- 회원이 “무엇이 좋아졌는지”를 이해하지 못한 사례가 있었나요?
- 현재 자료만으로 설명하기 어려운 항목은 무엇인가요?
- 어떤 형태의 요약 리포트가 있으면 상담에 도움이 될까요?

### 가설 3. 상담 품질 표준화

- 트레이너별 상담 방식 차이가 센터 운영에 영향을 주나요?
- 신규 트레이너가 상담 품질을 맞추는 데 어려움이 있나요?
- 센터장 입장에서 표준 리포트 양식이 있다면 어떤 항목을 꼭 넣고 싶나요?
- 리포트 품질을 확인할 책임자는 누구여야 하나요?

### 가설 4. 지불의사

- 월 구독형으로 제공된다면 어떤 조건에서 검토할 수 있나요? 가격은 `[확인필요]`로 두고 범위 반응만 확인합니다.
- 리포트당 과금 방식이라면 어떤 장단점이 있나요? 가격은 `[확인필요]`입니다.
- 비용을 지불하려면 어떤 증거가 먼저 필요할까요?
- 도입 결정권자는 누구이며, 결제 승인 과정은 어떻게 되나요?

### 가설 5. 트레이너 검토 기반 보고서

- AI가 작성한 초안을 트레이너가 검토해 확정하는 흐름을 어떻게 보나요?
- AI가 단독으로 결론을 내리는 방식과 비교하면 어떤 차이가 있나요?
- 트레이너가 반드시 수정하거나 확인해야 할 항목은 무엇인가요?
- 회원에게 “트레이너 검토 완료 리포트”라고 안내하면 신뢰에 도움이 될까요?

## 7. 솔루션 반응 질문

1. 운동 기록, RPE, 통증 반응, 생활습관, 트레이너 메모가 한 장의 보고서로 정리된다면 어디에 가장 먼저 쓰겠나요?
2. 상담 보고서가 재등록 상담에 도움이 되려면 어떤 문장과 그래프가 필요할까요?
3. 리포트 초안 생성 후 트레이너 확인 단계가 있다면 운영 부담은 어느 정도일까요?
4. 센터 내 여러 트레이너가 같은 양식을 쓰는 것에 대한 장점과 우려는 무엇인가요?
5. 파일럿에 참여한다면 최소 조건은 무엇인가요?

## 8. 파일럿 참여 확인 질문

- 2~4주 범위의 소규모 파일럿에 참여할 의향이 있나요? 기간은 `[확인필요]`입니다.
- 동의 기반, 비식별화, 최소수집 원칙에 동의할 수 있나요?
- 가상 샘플 리포트를 먼저 보고 피드백하는 방식이 적절한가요?
- 파일럿 담당자와 주간 피드백 시간을 지정할 수 있나요?

## 9. 마무리 질문

1. 오늘 이야기한 문제 중 가장 비용이 큰 문제는 무엇인가요?
2. 이 제품이 없어도 직접 해결할 수 있는 방법은 무엇인가요?
3. 바로 도입하기 어렵다면 가장 큰 이유는 무엇인가요?
4. 추가로 인터뷰하면 좋을 센터 대표, 센터장, 트레이너가 있나요?

## 10. 인터뷰 후 기록 규칙

- 실제 이름 대신 `인터뷰-001`, `센터-A`처럼 비식별 코드로 기록합니다.
- 구체적인 가격, 시장 규모, 시간 절감 수치는 검증 전까지 `[확인필요]`로 표시합니다.
- 제품 문구는 [비의료 경계 정책](../governance/non-medical-boundary.md)의 표현 원칙과 맞는지 확인합니다.
- 발견한 파일럿 후보는 [파일럿 센터 리스트 템플릿](pilot-center-list-template.md)에 별도로 정리합니다.
