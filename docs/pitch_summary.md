# LIGHT ONE 피치 요약

## 1. 30초 피치

LIGHT ONE은 PT샵과 필라테스 센터가 회원의 운동 수행, 통증 반응, 자세 변화, 생활습관 데이터를 구조화해 트레이너 검토용 상담 리포트를 생성하는 비의료 웰니스 SaaS입니다. 기존 자세분석 앱이 측정에 집중했다면, LIGHT ONE은 실제 PT 현장의 상담, 변화 설명, 재등록, 회원 유지 문제를 해결하는 데 집중합니다. 초기에는 리포트 MVP와 고객 인터뷰로 지불의사를 검증하고, 이후 컨디셔닝 방법론과 프리미엄 PT 브랜드 운영 시스템으로 확장합니다.

## 2. 1분 피치

PT 현장에서는 회원의 운동 기록, 통증 반응, 사진, 상담 메모가 여러 도구에 흩어져 있고, 트레이너는 재등록 상담 전 자료를 다시 정리해야 합니다. 회원도 자신의 변화가 어떻게 관리되는지 명확히 이해하기 어렵습니다.

LIGHT ONE은 이 문제를 해결하기 위해 회원 등록, 운동 목표, 통증 반응, 세션 기록, RPE, 트레이너 메모를 구조화하고, QS rule-based score와 AUTO/REVIEW/BLOCK 라우팅을 통해 트레이너가 검토할 수 있는 비의료 상담 리포트를 생성합니다. AI가 진단하거나 처방하는 서비스가 아니라, 트레이너의 상담 품질과 설명력을 높이는 Human-in-the-Loop 시스템입니다.

초기 고객은 프리미엄 PT샵 대표, 트레이너, 필라테스 스튜디오 대표, 교정운동 센터입니다. 현재 고객 traction, 유료 고객, 임상 검증, 특허 보유 주장은 하지 않으며, 인터뷰와 WTP 테스트로 실제 도입 가능성을 검증하겠습니다 [검증필요].

## 3. 3분 피치 아웃라인

| 순서 | 내용 | 핵심 메시지 |
|---:|---|---|
| 1 | 문제 | PT 상담 기록이 흩어져 있고 재등록 상담 자료가 부족하다. |
| 2 | 고객 | 프리미엄 PT샵, 트레이너, 필라테스 스튜디오, 교정운동 센터를 우선 검증한다. |
| 3 | 솔루션 | 운동 기록·통증 반응·RPE·메모를 구조화해 트레이너 검토용 리포트를 생성한다. |
| 4 | 안전 경계 | 의료 진단·질병 예측·자동 처방을 하지 않는 비의료 웰니스 SaaS이다. |
| 5 | 기술 | QS rule-based score, AUTO/REVIEW/BLOCK, 선택적 촬영 QC, PDF/웹 리포트로 MVP를 구성한다. |
| 6 | 비즈니스 모델 | 월 구독, 리포트당 과금, 셋업비, 교육·인증은 모두 WTP 검증 대상이다 [검증필요]. |
| 7 | 장기 전략 | SaaS + 컨디셔닝 방법론 + 프리미엄 브랜드 운영 시스템으로 확장한다. |
| 8 | 검증 계획 | 인터뷰, 샘플 리포트 피드백, WTP 테스트, 소규모 파일럿을 진행한다. |
| 9 | 요청 | MVP 검증과 고객 인터뷰를 위한 멘토링·파일럿 연결·초기 지원을 요청한다. |

## 4. 심사위원 Q&A 방어

| 질문 | 답변 |
|---|---|
| 의료기기 또는 진단 서비스 아닌가요? | 아닙니다. LIGHT ONE은 의료 진단, 치료, 처방, 질병 예측을 제공하지 않습니다. 운동 기록과 통증 반응을 구조화해 트레이너가 검토하는 비의료 상담 지원 서비스입니다. |
| AI가 통증 원인을 맞히나요? | 아닙니다. 통증 원인을 결정하지 않고, 회원이 입력한 통증 반응과 운동 전후 변화를 기록해 트레이너 확인이 필요한 신호로 분류합니다. |
| AI 정확도는 어떻게 검증하나요? | MVP에서는 AI 단독 판단이 아니라 rule-based QS score, 촬영 QC, AUTO/REVIEW/BLOCK 라우팅을 사용합니다. 임계값과 현장 적합성은 파일럿에서 [검증필요]입니다. |
| 고객이 돈을 낼 근거가 있나요? | 현재 유료 고객 보유 주장은 하지 않습니다. 인터뷰와 WTP 테스트로 상담 준비 시간, 재등록 상담 활용, 센터장 표준화 니즈가 지불의사로 이어지는지 검증하겠습니다. |
| 특허나 임상 검증이 있나요? | 현재 특허 보유 또는 임상 검증 보유 주장은 하지 않습니다. 필요 시 선행기술조사, 규제 검토, 전문가 검토를 진행하겠습니다 [검증필요]. |
| 경쟁사가 따라 하면 어떻게 하나요? | 단순 자세 분석이 아니라 상담 리포트 UX, 트레이너 검토 프로세스, 개인정보·비의료 안전 설계, 컨디셔닝 운영 방법론을 결합해 현장 운영 시스템으로 차별화합니다. |
| 개인정보 리스크는 어떻게 관리하나요? | 최소 수집, 비식별 코드, 공개 GitHub 내 실제 회원 데이터·사진·영상 금지, 원본 미디어 별도 저장, 접근 제어, 삭제 요청 처리를 원칙으로 합니다. |

## 5. 리스크 대응 표

| 리스크 | 영향 | 대응 |
|---|---|---|
| 의료기기 오해 | 심사·영업·규제 리스크 증가 | 비의료 고지, 금지 표현표, Human-in-the-Loop, 진단·치료·처방 제외 |
| AI 정확도 공격 | 기술 신뢰도 저하 | MVP는 rule-based score와 QC, 트레이너 검토 중심으로 제한 |
| 고객 지불의사 부족 | 수익모델 검증 실패 | WTP 인터뷰, 리포트당 과금 테스트, 유료 PoC 조건 확인 |
| 입력 부담 | 현장 사용성 저하 | 필수 입력 최소화, 트레이너 메모 간소화, 리포트 자동 정리 |
| 개인정보 노출 | 신뢰·법적 리스크 | 실제 데이터 공개 금지, 원본 미디어 별도 저장, 접근 제어, 삭제 절차 |
| 과장된 traction 주장 | 심사 신뢰 하락 | 확인되지 않은 고객 traction, 유료 고객, 임상 검증, 특허 주장은 제외 |
| 브랜드 확장 과속 | 비용·운영 부담 증가 | SaaS MVP → 파일럿 → WTP → 방법론 → 교육·제휴 순서로 진행 |
