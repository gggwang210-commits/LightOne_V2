# 개인정보 보호 체크리스트

이 체크리스트는 LIGHTONE V2의 파일럿, MVP 운영, 상담 보고서 생성 과정에서 개인정보와 민감한 운동·통증 기록을 안전하게 다루기 위한 기준입니다. 사업 방향은 [루트 README](../../README.md#6-데이터-아키텍처), MVP 입력 항목은 [Django README](../../lightone_v2_django/README.md#주요-기능)를 함께 참고합니다.

## 1. 수집 전 확인

- [ ] 회원에게 수집 목적, 항목, 보관 기간, 삭제 요청 방법을 설명했는가?
- [ ] 파일럿 또는 테스트 참여 동의를 별도로 받았는가?
- [ ] 의료 진단 목적이 아니라 비의료 운동 상담 기록 목적임을 고지했는가?
- [ ] 필수 항목과 선택 항목을 구분했는가?
- [ ] 공개 저장소, 데모 데이터, 발표 자료에 실명·연락처·얼굴 이미지가 포함되지 않는가?

## 2. 최소 수집 원칙

| 구분 | 권장 수집 | 피해야 할 수집 |
|------|-----------|----------------|
| 회원 정보 | 회원 ID, 연령대, 성별, 운동 목표 | 주민등록번호, 상세 주소, 불필요한 연락처 |
| 운동 기록 | 운동명, 세트, 반복, RPE, 통증 반응 | 의료 진단명, 처방전, 의무기록 |
| 이미지·영상 | 촬영 품질 확인용 Media ID, 촬영 조건 | 동의 없는 원본 얼굴·신체 이미지 공유 |
| 상담 메모 | 운동 상담에 필요한 관찰 기록 | 사생활, 가족력, 불필요한 민감 정보 |

## 3. 저장·접근 관리

- [ ] 운영 DB와 개발·데모 DB를 분리했는가?
- [ ] 원본 이미지와 분석 지표를 분리 저장하는가?
- [ ] 관리자, 트레이너, 개발자 권한을 구분했는가?
- [ ] 접근 로그 또는 변경 이력을 남길 수 있는가?
- [ ] 백업 파일과 `.env` 파일이 Git에 올라가지 않도록 확인했는가?
- [ ] 샘플 데이터는 익명 또는 더미 데이터만 사용하는가?

## 4. 파일럿 운영 체크

- [ ] 파일럿 시작 전 [파일럿 검증 계획](../validation/pilot-validation-plan.md)의 동의·중단 기준을 확정했는가?
- [ ] 센터 담당자에게 개인정보 처리 범위를 설명했는가?
- [ ] 통증 악화, 세션 중단, 의료 상담 안내 등 예외 상황 기록 절차가 있는가?
- [ ] 파일럿 종료 후 보관·삭제·익명화 처리 방법을 안내했는가?

## 5. 실행 환경 체크

Windows에서 로컬 MVP를 실행할 때도 실제 회원 데이터 대신 더미 데이터를 사용합니다. 실행 절차는 [Windows 실행 가이드](../execution-guide/windows-run-guide.md)와 [기존 RUN_WINDOWS](../../lightone_v2_django/RUN_WINDOWS.md)를 따르되, 화면 캡처나 공유 전에는 개인정보가 노출되지 않는지 확인합니다.

## 6. 출시 전 확인 필요

- [확인필요] 개인정보 처리방침 및 이용약관의 법률 검토
- [확인필요] 민감정보 해당 여부와 동의 양식 검토
- [확인필요] 위탁 처리, 클라우드 저장, 국외 이전 여부 검토
- [확인필요] 보관 기간, 파기 절차, 삭제 요청 SLA 확정
