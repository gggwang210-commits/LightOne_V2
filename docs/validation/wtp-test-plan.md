# WTP 지불의사 테스트 계획

이 문서는 LIGHTONE V2의 월 구독형 또는 리포트당 과금 모델에 대한 지불의사를 검증하기 위한 테스트 계획입니다. 가격과 전환율은 검증 전이므로 모든 수치를 `[확인필요]`로 표시합니다. 관련 인터뷰 질문은 [트레이너·센터 고객 인터뷰 질문지](trainer-interview-script.md), 기록 양식은 [고객 발견 인터뷰 로그 템플릿](customer-discovery-log-template.md)을 사용합니다.

## 1. 테스트 목적

- 비의료 PT 상담 보고서 SaaS가 실제 비용 지불 대상인지 확인합니다.
- 월 구독, 리포트당 과금, 혼합 모델 중 초기 수용성이 높은 방식을 찾습니다.
- 구매자와 사용자가 다른 경우 의사결정 구조를 파악합니다.
- 가격 반응보다 “어떤 증거가 있으면 결제할 수 있는지”를 우선 확인합니다.

## 2. 검증 가설

| 번호 | 가설 | 확인 방법 |
|------|------|-----------|
| 1 | 상담 준비 시간 절감이 비용 지불 근거가 된다 | 시간 사용 사례와 비용 인식 질문 |
| 2 | 재등록 상담 자료 부족이 리포트 구매 니즈로 이어진다 | 샘플 리포트 반응 확인 |
| 3 | 센터장은 상담 품질 표준화에 예산을 배정할 수 있다 | 결정권자·예산 항목 확인 |
| 4 | 월 구독 또는 리포트당 과금 선호가 고객군별로 다르다 | 과금 방식 비교 질문 |
| 5 | 트레이너 검토 기반 보고서가 AI 단독 산출물보다 도입 장벽이 낮다 | 신뢰·책임·운영 질문 |

## 5. 질문 설계

### 5.1 현재 비용 인식

1. 상담 준비와 기록 정리에 들어가는 시간을 비용으로 본다면 어느 정도 부담인가요? `[확인필요]`
2. 재등록 상담 자료를 만들기 위해 별도 디자인, 문서, 수기 정리를 해본 적이 있나요?
3. 현재 사용하는 도구에 비용을 지불하고 있나요? 금액은 범위만 기록하고 `[확인필요]`로 표시합니다.

### 5.2 과금 방식 비교

1. 월 구독 방식의 장점과 우려는 무엇인가요?
2. 리포트당 과금 방식의 장점과 우려는 무엇인가요?
3. 센터 단위 계정과 트레이너 개인 계정 중 어느 방식이 더 자연스러운가요?
4. 어떤 조건이 충족되면 유료 파일럿을 검토할 수 있나요?

### 5.3 구매 트리거

1. 상담 준비 시간이 줄었다는 증거가 있으면 결제 검토가 가능한가요?
2. 재등록 상담에서 보고서를 사용한 사례가 있으면 가치가 커지나요?
3. 트레이너별 상담 양식이 표준화되면 센터 운영에 도움이 되나요?
4. 회원에게 제공되는 보고서가 센터 브랜드 경험에 기여한다고 보나요?

### 5.4 반대·우려 확인

1. 지불을 망설이게 하는 가장 큰 이유는 무엇인가요?
2. 입력 시간이 늘어나면 어느 수준까지 허용 가능한가요? `[확인필요]`
3. 회원 동의 절차가 추가되면 운영 부담은 어느 정도인가요?
4. AI 초안에 대한 책임 소재를 어떻게 안내해야 안심할 수 있나요?

## 6. 성공·중단 기준

| 기준 | 통과 신호 | 중단 또는 수정 신호 |
|------|-----------|---------------------|
| 문제 강도 | 최근 사례와 반복성이 확인됨 | “가끔 있으면 좋다” 수준에 머묾 |
| 지불의사 | 조건부 결제 또는 유료 파일럿 논의 가능 | 무료 사용 외 관심 없음 |
| 과금 방식 | 선호 모델과 이유가 명확함 | 어떤 방식도 납득하지 못함 |
| 도입 장벽 | 동의·입력·검토 흐름을 수용 | 운영 부담이 가치보다 큼 |
| 비의료 신뢰 | 트레이너 검토 기반 설명을 선호 | 자동 산출물 책임에 대한 우려가 큼 |

- 가격 수치보다 구매 조건과 예산 소유자를 우선 기록합니다.

- 검증 전 수치는 README와 제출 문서에 반영하지 않거나 `[확인필요]`로 남깁니다.
