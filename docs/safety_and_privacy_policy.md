# LIGHT ONE 안전 및 개인정보 정책

## 1. 비의료 고지

LIGHT ONE은 비의료 웰니스·운동 상담 지원 서비스이다. 서비스는 회원의 운동 기록, RPE, 통증 반응, 트레이너 메모, 촬영 QC 정보를 구조화하고 트레이너 검토용 상담 리포트 초안을 제공한다.

LIGHT ONE은 다음을 제공하지 않는다.

- 의료 진단
- 치료 또는 재활 치료
- 질병 위험 예측
- 통증 원인 확정
- 임상 의사결정 지원
- AI 단독 판단에 따른 자동 처방

## 2. 금지 표현

| 금지 표현 | 금지 사유 |
|---|---|
| 진단 | 의료 행위로 오해될 수 있음 |
| 치료 | 치료 효과 주장으로 오해될 수 있음 |
| 처방 | 의학적 처방 또는 자동 운동 처방으로 오해될 수 있음 |
| 재활 치료 | 의료·재활 영역으로 오해될 수 있음 |
| 통증 원인 분석 | 의학적 인과 판단으로 오해될 수 있음 |
| 질병 위험도 예측 | 질병 예측 기능으로 오해될 수 있음 |
| AI가 판단 | 사람 검토 없는 자동 의사결정으로 오해될 수 있음 |
| 완치·개선 보장 | 효과 보장 주장으로 오해될 수 있음 |

## 3. 안전 대체 표현

| 위험 표현 | 안전 대체 표현 |
|---|---|
| 진단 | 기록, 확인, 상담 참고 |
| 치료 | 관리, 컨디셔닝, 운동 상담 |
| 처방 | 운동 제안, 프로그램 조정 제안 |
| 재활 | 컨디셔닝, 운동 상담 플랜 |
| 통증 원인 분석 | 통증 반응 기록 |
| 위험도 판정 | 주의 신호, 트레이너 확인 필요 |
| AI가 판단 | 트레이너가 검토 |
| 의료 수준 분석 | 비의료 웰니스 참고 리포트 |

## 4. 데이터 최소 수집 원칙

- 서비스 제공과 파일럿 검증에 필요한 최소 데이터만 수집한다.
- 주민등록번호, 금융정보, 불필요한 건강정보 등 고위험 정보는 기본 수집 대상에서 제외한다.
- 이름, 연락처, 이메일 등 직접 식별자는 가능하면 별도 승인된 도구에 분리 저장한다.
- 운영 로그, 오류 로그, 분석 이벤트에는 개인정보가 남지 않도록 마스킹 또는 제외 처리한다.
- 가격, 시장 규모, 성과 수치, 임계값은 검증 전까지 `[검증필요]`로 표시한다.

## 5. 공개 GitHub 내 실제 회원 데이터 금지

공개 GitHub 저장소에는 실제 회원의 이름, 연락처, 이메일, 사진 파일명, 상담 기록 원문, 통증 기록 원문, 센터 계약 정보 등 식별 가능 정보를 커밋하지 않는다.

## 6. 공개 저장소 내 실제 사진/영상 금지

실제 회원의 사진, 영상, 음성, 얼굴이 포함된 캡처, 촬영 원본, 업로드 미디어는 공개 저장소에 저장하지 않는다. 데모가 필요하면 합성 이미지, 익명화 이미지, 또는 권리와 동의가 명확한 샘플만 사용한다.

## 7. 합성 또는 익명화 샘플 데이터만 사용

- 데모 데이터는 `회원-A`, `센터-A`, `인터뷰-001` 같은 비식별 코드로 작성한다.
- 실제 사례를 외부 제출 문서에 넣어야 하는 경우 직접 식별자를 제거하고 범주형·요약형으로 변환한다.
- 재식별 가능성이 있으면 공개 자료에 포함하지 않는다.

## 8. 원본 미디어 별도 저장

- 운영 또는 파일럿 중 수집된 원본 사진·영상은 접근 권한이 통제된 별도 스토리지에 보관한다.
- 원본 미디어는 공개 GitHub, 공개 문서, 공유 링크에 업로드하지 않는다.
- 얼굴 블러, 얼굴 제외 촬영, 촬영 품질 QC만 저장하는 방식 등 원본 저장 최소화 옵션을 우선 검토한다 [검증필요].

## 9. 접근 제어

- 개인정보와 원본 미디어 접근 권한은 업무상 필요한 담당자로 제한한다.
- 접근자, 접근 목적, 반출 여부를 기록한다.
- 운영자, 트레이너, 관리자 권한은 역할별로 분리한다.
- 외부 공유 전 비식별화 상태와 민감정보 포함 여부를 재검토한다.

## 10. 보관 기간

- 수집 목적별 보관 기간을 사전에 안내한다.
- 파일럿 데이터 보관 기간은 파일럿 계약 또는 동의서에서 정한다 [검증필요].
- 목적 달성 후에는 삭제 또는 비식별 보관 여부를 결정하고 기록한다.
- 백업 데이터의 보관·파기 기준도 함께 정의한다 [검증필요].

## 11. 삭제 요청 처리

- 회원 또는 센터가 열람, 정정, 삭제를 요청할 수 있는 채널을 마련한다.
- 요청 접수일, 담당자, 처리 기한, 처리 결과를 기록한다.
- 삭제 요청 처리 시 운영 DB, 파일 스토리지, 백업, 로그, 외부 연동 도구의 잔존 데이터를 함께 확인한다.
- 법적 보관 의무가 있거나 즉시 삭제가 어려운 경우 사유와 예상 처리 일정을 안내한다 [검증필요].

## 12. Human-in-the-Loop 트레이너 검토

- 시스템 리포트는 초안이며, 회원에게 전달하기 전 트레이너가 최종 검토한다.
- `AUTO`는 일반 상담 보고서 생성 가능 상태이나 트레이너 확인 후 전달한다.
- `REVIEW`는 통증 반응 또는 품질 경계값이 있어 트레이너 확인이 필요하다.
- `BLOCK`은 세션 중단 권고 신호이며 의료 진단이 아니다. 필요 시 의료 전문가 상담을 안내한다.
