# LIGHT ONE V2 기능 개발 가이드

이 문서는 LIGHT ONE V2의 다음 기능을 추가·수정하는 방법을 설명합니다.
비전공자도 이해할 수 있도록 각 단계를 쉽게 설명합니다.

---

## 개발 우선순위 로드맵

| 단계 | 기능 | 난이도 | 예상 소요 시간 |
|------|------|--------|--------------|
| **1단계 (현재)** | 세션 입력 + QS 자동 계산 + 위험 라우팅 | 기초 | 완료 |
| **2단계** | Chart.js 변화 추세 대시보드 | 초급 | 1~2일 |
| **3단계** | 트레이너 검토(REVIEW) 승인 기능 | 초급 | 1일 |
| **4단계** | PDF 비의료 상담 리포트 출력 | 중급 | 2~3일 |
| **5단계** | XGBoost/LightGBM 예측 모델 연동 | 고급 | 1주일+ |

## 2단계: 변화 추세 대시보드 추가하기

### 무엇을 만드나요?
회원별 QS 점수와 통증(NRS) 변화를 그래프로 보여주는 화면입니다.

### 어떤 파일을 수정하나요?

**① `lightone/views.py`에 데이터 준비 코드 추가**

`dashboard` 함수 안에 아래 코드를 추가합니다:
```python
# 최근 10개 세션의 QS 점수 데이터 준비
from lightone.models import PTSession
import json

sessions = PTSession.objects.filter(member=member).order_by('session_date')[:10]
qs_labels = [s.session_date.strftime('%m/%d') for s in sessions]
qs_data   = [s.qs_score for s in sessions]
nrs_data  = [s.pain_nrs for s in sessions]

context['qs_labels'] = json.dumps(qs_labels)
context['qs_data']   = json.dumps(qs_data)
context['nrs_data']  = json.dumps(nrs_data)
```

**② `templates/lightone/dashboard.html`에 그래프 코드 추가**

`</body>` 태그 바로 위에 아래 코드를 추가합니다:
```html
<!-- Chart.js 라이브러리 불러오기 -->
<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
<canvas id="qsChart" width="400" height="150"></canvas>
<script>
  const ctx = document.getElementById('qsChart').getContext('2d');
  new Chart(ctx, {
    type: 'line',
    data: {
      labels: {{ qs_labels|safe }},
      datasets: [
        { label: 'QS 점수', data: {{ qs_data|safe }}, borderColor: '#0E9488', tension: 0.3 },
        { label: '통증(NRS)', data: {{ nrs_data|safe }}, borderColor: '#EF4444', tension: 0.3 }
      ]
    }
  });
</script>
```

## 3단계: REVIEW 승인 기능 추가하기

### 무엇을 만드나요?
REVIEW 상태인 세션을 트레이너가 확인하고 "승인" 또는 "BLOCK으로 변경" 버튼을 누를 수 있는 기능입니다.

### 어떤 파일을 수정하나요?

**① `lightone/models.py`에 승인 상태 필드 추가**
```python
# PTSession 모델 안에 아래 필드 추가
trainer_approved = models.BooleanField(default=False, verbose_name='트레이너 승인 여부')
trainer_note     = models.TextField(blank=True, verbose_name='트레이너 메모')
```

**② 수정 후 반드시 아래 명령어 실행 (DB 반영)**
```bash
python manage.py makemigrations
python manage.py migrate
```

## 4단계: PDF 리포트 출력 기능 추가하기

### 무엇을 만드나요?
회원에게 전달할 수 있는 비의료 상담 리포트를 PDF로 출력하는 기능입니다.

### 패키지 설치
```bash
pip install reportlab
```

### 핵심 코드 예시 (`lightone/views.py`에 추가)
```python
from reportlab.pdfgen import canvas
from django.http import HttpResponse

def download_report_pdf(request, session_id):
    session = PTSession.objects.get(id=session_id)
    
    response = HttpResponse(content_type='application/pdf')
    response['Content-Disposition'] = f'attachment; filename="lightone_report_{session_id}.pdf"'
    
    p = canvas.Canvas(response)
    p.setFont("Helvetica-Bold", 16)
    p.drawString(100, 800, "LIGHT ONE - 비의료 운동 상담 참고 자료")
    p.setFont("Helvetica", 12)
    p.drawString(100, 760, f"회원명: {session.member.user.get_full_name()}")
    p.drawString(100, 740, f"세션 날짜: {session.session_date}")
    p.drawString(100, 720, f"QS 점수: {session.qs_score}")
    p.drawString(100, 700, f"위험 라우팅: {session.risk_route}")
    p.drawString(100, 660, "※ 본 자료는 의료적 진단·치료·처방을 대체하지 않습니다.")
    p.showPage()
    p.save()
    
    return response
```
