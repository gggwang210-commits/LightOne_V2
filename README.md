# LIGHTONE_V2

> AI 체형분석을 단독 상품으로 판매하는 프로젝트가 아니라, **비의료 PT 상담 리포트 SaaS + 기능적 컨디셔닝 OS + 교육·인증 IP**로 확장하기 위한 사업 제작 전략 저장소입니다.

작성 기준일: 2026-06-30  
프로젝트 상태: 전략 전환 / 사업 제작 단계  
핵심 키워드: `PT 상담 리포트`, `기능적 컨디셔닝`, `QS`, `JATC`, `Human-in-the-loop`, `비의료 웰니스`, `모두의 창업 2기`

---

## 1. Executive Summary

LIGHTONE_V2의 핵심 방향은 다음과 같습니다.

1. **측정 솔루션 경쟁에서 벗어난다.**  
   체형분석 키오스크, 자세분석 앱, 피트니스 CRM 시장은 이미 경쟁자가 많습니다. LIGHTONE_V2는 “AI로 체형을 측정한다”가 아니라, PT 현장의 상담·기록·재등록 문제를 해결하는 방향으로 전환합니다.

2. **송광일의 현장 방법론을 무형 IP로 자산화한다.**  
   핵심 자산은 카메라 기술 자체가 아니라, 기능근육 파악 → 약한 기능 회복 → 컨디셔닝 기초 위 트레이닝으로 이어지는 의사결정 프로토콜입니다.

3. **초기 제품은 비의료 PT 상담 리포트 SaaS로 정의한다.**  
   의료 진단, 치료, 재활, 처방을 하지 않고, 트레이너가 회원 상담에 활용할 수 있는 리포트 초안과 주의 신호를 제공합니다.

4. **사업 확장은 교육·인증 + B2B 근골격 예방 모델로 설계한다.**  
   모두의 창업 2기 관점에서는 단순 앱보다 `컨디셔닝 OS + 트레이너 교육·인증 + 기업 B2B 파일럿` 조합이 확장성과 수익모델을 설명하기 유리합니다.

## 2. One-line Definition

**LIGHTONE_V2는 PT샵·헬스장·필라테스 센터가 회원의 운동 수행, 통증 반응, 자세 변화, 생활습관 데이터를 구조화해 재등록 상담과 컨디셔닝 지도에 활용할 수 있도록 돕는 비의료 운동상담 리포트 SaaS입니다.**

---

## 3. Why V2?

초기 LIGHT ONE은 AI 체형분석, 촬영 품질 확인, 조명 정규화, 카메라 캘리브레이션 중심으로 설명되었습니다.

그러나 조사 결과, 시장과 심사 관점에서 다음 리스크가 확인되었습니다.

| 구분 | 기존 방향 | 리스크 | V2 전환 방향 |
|---|---|---|---|
| 시장 | AI 체형분석 | 키오스크·앱 경쟁 심함 | 상담 리포트 + 컨디셔닝 OS |
| 기술 | 포즈추정·측정 | 오픈소스/SDK 상향평준화 | 촬영 QC를 신뢰도 보조 레이어로 배치 |
| 규제 | 통증·체형 원인 분석 | 의료기기·의료행위 오인 가능성 | 비의료 운동상담 보조 |
| 사업성 | D2C 앱 | 리텐션·마케팅 비용 부담 | B2B 센터·트레이너·기업 파일럿 |
| 심사 | AI 측정 서비스 | 차별성 입증 부담 | 방법론 IP + 실행 로드맵 |

## 4. Core Customer Problem

PT 현장에서는 다음 문제가 반복됩니다.

- 회원의 운동 기록, 촬영 기록, 통증 반응, 상담 메모가 카카오톡, 사진첩, 메모장, 엑셀, 인바디 결과지 등에 분산됩니다.
- 트레이너는 상담 전 자료 정리에 시간을 씁니다.
- 회원 변화가 있어도 재등록 상담 때 설명할 객관적 자료가 부족합니다.
- 원장은 트레이너별 상담 품질을 표준화하기 어렵습니다.
- 촬영 각도, 조명, 거리 차이로 전후 비교 신뢰도가 흔들립니다.

따라서 LIGHTONE_V2는 **측정 결과 자체**보다 **상담 가능한 언어로 정리된 리포트와 표준화된 컨디셔닝 의사결정 흐름**을 제공합니다.

## 6. Product Modules

### 6.1 상담 입력 모듈

- 회원 기본정보
- 운동 목표
- 주요 불편감 부위
- 세션별 운동 기록
- RPE
- 통증 반응
- 생활습관 체크
- 트레이너 메모

### 6.2 QS Engine

`QS(Quality Score, 운동 수행 품질지수)`는 세트 단위 운동 수행 품질을 요약하는 지표입니다.

초기 구성요소:

- Form Accuracy, 폼 정확도
- Rep Achievement Rate, 반복수 달성률
- Rest Compliance, 휴식 준수 여부
- Pain Response, 통증 반응

초기 라우팅:

| 상태 | 조건 예시 | 의미 |
|---|---|---|
| AUTO | QS 양호, 특이사항 없음 | 일반 상담 리포트 생성 |
| REVIEW | QS 중간 또는 통증 반응 존재 | 트레이너 확인 필요 |
| BLOCK | 통증 급증, 수행 중단, 위험 신호 | 세션 중단 또는 전문가 상담 권고 문구 |

> 주의: QS는 의료적 판단 지표가 아니라 운동상담 참고 지표입니다.

### 6.3 JATC Engine

`JATC(Joint Achievement & Tracking Composite, 통합 상태지수)`는 통증 반응, 자세 변화, 근기능, 생활습관을 하나의 상담용 상태 흐름으로 정리하기 위한 지표입니다.

초기 구성:

- Pain, 통증 반응
- Posture, 자세 변화
- Function & Training, 근기능·운동 수행
- Lifestyle, 생활습관

> [확인필요] JATC 가중치와 임계값은 파일럿 데이터와 전문가 검토 후 확정합니다.

### 6.4 Report Generator

상담 리포트 유형:

1. **Basic Report**: 촬영·입력 품질이 양호한 기본 리포트
2. **Review Report**: 트레이너 확인이 필요한 리포트
3. **Retake Report**: 촬영 조건 미달 또는 입력 품질 부족으로 재촬영·재입력이 필요한 리포트

### 6.5 Trainer Review Flow

AI는 최종 판단자가 아닙니다.

흐름:

```text
데이터 입력
  → QS/JATC 규칙 기반 산출
  → AUTO / REVIEW / BLOCK 라우팅
  → 트레이너 검토
  → 상담 리포트 확정
  → 회원 상담 및 재등록 관리 활용
```

## 7. Technology Layer

기존 LIGHT ONE 기술 자료는 V2에서도 버리지 않습니다. 다만 메인 상품이 아니라 **리포트 신뢰도 보조 레이어**로 재배치합니다.

### 7.1 Camera Calibration

역할:

- 촬영 품질 PASS / CHECK / FAIL 분류
- 카메라 내부 파라미터 계산
- 왜곡 계수 계산
- 재투영 오차 계산
- 보정 전후 비교 이미지 생성

### 7.2 Lighting Normalization

역할:

- Gray-world white balance
- CLAHE 기반 밝기 정규화
- 평균 밝기 정규화
- 조명 균일도 계산
- 그림자 비율 계산
- PNG 및 JSON 리포트 생성

### 7.3 Center QC

역할:

- 촬영 환경 기준 확인
- 품질 게이트 판정
- 스케일 기준 확인
- 촬영 조건 로그화

### 7.4 Pose Estimation Expansion

후속 단계에서 다음 기술을 연동할 수 있습니다.

- MediaPipe
- OpenPose
- MoveNet
- Apple Vision Body Pose

단, 포즈추정은 이미 범용화된 기술이므로 해자는 아닙니다. LIGHTONE_V2의 해자는 **현장 데이터 구조화, 컨디셔닝 방법론, 상담 리포트 UX, 트레이너 교육·인증 체계**에 있습니다.

## 8. Data Architecture

### 8.1 3-layer Data Model

```text
Member Layer
  - Member_ID
  - 성별, 연령대, 운동 목표, 운동 경험
  - 주요 불편감 부위
  - 동의 상태

Session Layer
  - Session_ID
  - 운동명, 세트, 반복, 중량, 휴식
  - RPE, 통증 반응, 트레이너 메모
  - 촬영 조건, 촬영 품질 상태

Indicator Layer
  - QS
  - JATC
  - REVIEW/BLOCK 신호
  - 상담 우선순위
  - 리포트 생성 상태
```

### 8.2 원본 영상과 분석 결과 분리

- 원본 이미지/영상은 별도 저장소에 보관합니다.
- 분석 테이블에는 Media_ID, Session_ID, 촬영 조건, 추출 지표, 품질 상태만 저장합니다.
- 실제 회원 이미지와 민감 건강정보는 저장소에 업로드하지 않습니다.

- 질병 예측
- 치료 효과 판단

- 환자 상태 평가
- 자동 운동 처방
- 재활 효과 검증

| 금지 표현 | 안전한 대체 표현 |
|---|---|
