# Dashboard Component Library

All components must avoid diagnosis, treatment, prescription, pain-cause determination, and disease prediction wording.

## MetricCard

- Purpose: Provide a reusable dashboard block for Metric card in member, trainer, or admin workflows.
- Props / data fields: `id`, `title`, `status`, `summary`, `updatedAt`, `items[]`, `ctaLabel`, `ctaHref`, `privacyLevel`.
- Role usage: Member when the content is self-explanatory; Trainer when review or feedback is needed; Admin when aggregated operations or permission monitoring is needed.
- Safe wording notes: Use “기록”, “참고 지표”, “검토 필요”, “컨디셔닝 제안”. Do not use “진단”, “치료 효과”, “통증 원인 분석”, “처방”.
- Empty state copy: “아직 표시할 기록이 없습니다. 다음 세션 후 업데이트됩니다.”
- Error state copy: “정보를 불러오지 못했습니다. 잠시 후 다시 확인해 주세요.”
