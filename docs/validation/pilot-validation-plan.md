# 파일럿 검증 계획

이 문서는 LIGHTONE V2의 초기 파일럿에서 “PT 상담 보고서 SaaS가 실제 현장 문제를 줄이는가?”를 검증하기 위한 1차 계획입니다. 제품 가설은 [루트 README §9 시장 진출 전략](../../README.md#9-시장-진출-전략), MVP 기능은 [Django README](../../lightone_v2_django/README.md), 실행 절차는 [Windows 실행 가이드](../execution-guide/windows-run-guide.md)를 참조합니다.

## 1. 검증 목표

- 트레이너의 상담 준비 시간이 줄어드는지 확인한다.
- 회원이 운동 변화와 통증 반응을 더 쉽게 이해하는지 확인한다.
- QS/JATC 기반 AUTO / REVIEW / BLOCK 라우팅이 현장 검토 우선순위에 도움이 되는지 확인한다.
- 비의료 표현과 개인정보 처리 절차가 파일럿 환경에서 지켜지는지 확인한다.

## 2. 파일럿 범위

| 항목 | 1차 목표 |
|------|----------|
| 센터 수 | 1~3곳 |
| 참여 회원 | 센터별 5~10명, 총 10~30명 |
| 운영 기간 | 2~4주 |
| 데이터 | 운동 기록, RPE, 통증 반응, 트레이너 메모, 보고서 피드백 |
| 제외 | 의료 진단, 치료 효과 검증, 임상 연구 |

## 3. 핵심 지표

| 지표 | 측정 방법 | 목표 기준 |
|------|-----------|-----------|
| 상담 준비 시간 | 기존 방식 대비 보고서 준비 시간 비교 | 20% 이상 단축 가설 |
| 보고서 생성 성공률 | 입력 완료 후 보고서 생성 비율 | 80% 이상 가설 |
| 트레이너 만족도 | 5점 척도 설문 | 평균 4.0 이상 가설 |
| 회원 이해도 | 상담 후 5점 척도 설문 | 평균 4.0 이상 가설 |
| REVIEW/BLOCK 적절성 | 트레이너 사후 평가 | 오탐·누락 사례 기록 |
| 재등록 상담 활용 | 상담에서 보고서 사용 여부 | 정성 사례 확보 |

## 4. 운영 절차

1. 센터 대표와 파일럿 범위, 기간, 책임자를 확정합니다.
2. [개인정보 보호 체크리스트](../governance/privacy-checklist.md)를 적용해 동의와 데이터 항목을 확정합니다.
3. 담당 트레이너에게 비의료 표현 기준을 설명합니다. 기준 문서는 [비의료 경계 정책](../governance/non-medical-boundary.md)입니다.
4. MVP를 설치하고 더미 데이터로 리허설합니다. Windows 환경은 [기존 RUN_WINDOWS](../../lightone_v2_django/RUN_WINDOWS.md)도 함께 확인합니다.
5. 실제 파일럿 세션에서 운동 기록, RPE, 통증 반응, 메모를 입력합니다.
6. 보고서를 생성하고 트레이너가 최종 검토 후 상담에 활용합니다.
7. 주간 회고에서 문제 사례, 누락 항목, 문구 리스크를 정리합니다.
8. 종료 후 지표와 인터뷰를 정리해 다음 제품 개선 항목을 결정합니다.

## 5. 중단 및 예외 기준

- 참여자가 동의 철회를 요청하면 해당 데이터 수집을 중단합니다.
- 통증 악화 또는 이상 반응이 있으면 운동 세션을 중단하고 전문가 상담을 안내합니다.
- 개인정보 노출 가능성이 확인되면 파일럿 운영을 멈추고 원인과 조치 내역을 기록합니다.
- 트레이너가 보고서 문구를 의료 진단으로 오해할 수 있다고 판단하면 즉시 문구를 수정합니다.

## 6. 산출물

- 파일럿 참여 동의 기록
- 센터별 운영 로그와 이슈 목록
- 지표 요약표와 정성 인터뷰 메모
- 보고서 샘플 3종
- 모두의창업 제출에 활용 가능한 검증 요약. 제출용 문구는 [모두의창업 요약](../submission/modu-startup-summary.md)을 기준으로 정리합니다.
