# LIGHT ONE 기술 로드맵

## Phase 0~7 로드맵

| Phase | Goal | Expected output | Validation method | Risk |
|---:|---|---|---|---|
| 0. 포지셔닝·안전 경계 고정 | 비의료 PT 상담 리포트 SaaS로 범위 고정 | 제품 정의, 금지 표현표, 개인정보 원칙, MVP 범위 문서 | 문서 리뷰, 진단·치료·처방 표현 제거 체크 | 의료기기 오해, 과장 주장, 불명확한 고객 정의 |
| 1. 문제 검증 | 실제 센터·트레이너 문제가 존재하는지 확인 | 인터뷰 로그, 샘플 리포트 피드백, WTP 질문 결과 | 프리미엄 PT샵 대표·트레이너·필라테스·교정운동 센터 인터뷰 | 고객 traction으로 오해되는 미검증 발언, 유도 질문 |
| 2. 리포트 MVP | 회원·세션·통증 반응 데이터를 입력하고 리포트 생성 | 입력폼, QS rule-based score, AUTO/REVIEW/BLOCK, PDF/웹 리포트 | 합성 데이터 데모, 트레이너 사용성 피드백 | 입력 부담, 리포트 품질 부족, 임계값 [검증필요] |
| 3. 촬영 QC 선택 기능 | 촬영 품질을 상담 참고 정보로 표시 | 각도·조명·거리·전신 포함 여부 QC 결과 | 합성/익명화 샘플로 PASS/CHECK/FAIL 확인 | 원본 미디어 개인정보 리스크, 포즈 추정 정확도 오해 |
| 4. 파일럿 운영 | 동의 기반 소규모 현장 검증 | 센터 1~3곳 가설 [검증필요], 운영 로그, 주간 회고, 이슈 목록 | 2~4주 파일럿, 상담 준비 시간·리포트 사용성·REVIEW/BLOCK 적절성 기록 | 실제 회원 데이터 관리, 운영 중단 기준 미흡, 무료 사용에 그칠 위험 |
| 5. WTP·유료 PoC 검증 | 비용 지불 조건과 과금 방식을 확인 | 월 구독/리포트당 과금 반응, 필요한 증거, 구매자·사용자 구조 | 가격 범위 인터뷰, 조건부 결제 또는 유료 파일럿 논의 | 가격 수치 미검증, 유료 고객 보유로 과장 표현 위험 |
| 6. 운영 시스템화 | 센터 운영에 맞는 권한·검토·리포트 흐름 정리 | 관리자/트레이너 역할, 접근 제어, 리포트 승인, 삭제 요청 프로세스 | 파일럿 피드백 반영, 개인정보 체크리스트 검수 | 권한 설계 미흡, 삭제 요청 처리 누락, 트레이너 검토 병목 |
| 7. 브랜드·확장 검증 | SaaS를 컨디셔닝 방법론·교육·제휴 모델로 확장 가능성 확인 | LIGHT ONE 컨디셔닝 프로토콜 초안, 교육·인증 가설, 제휴 브랜드 제안서 | 센터장·트레이너 대상 확장 니즈 인터뷰, 파일럿 후 재사용 의향 확인 | 방법론 검증 부족, 교육·인증 수요 [검증필요], 브랜드 확장 과속 |

## 운영 원칙

- 각 Phase는 이전 Phase의 검증 결과를 근거로 진행한다.
- 고객 traction, 유료 고객, 임상 검증, 특허 보유 등 확인되지 않은 주장은 사용하지 않는다.
- 불확실한 가격, 성능, 전환율, 임계값, 법률 판단은 `[검증필요]`로 표시한다.
