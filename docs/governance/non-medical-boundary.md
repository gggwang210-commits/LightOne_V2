# 비의료 서비스 경계 정책

LIGHTONE V2는 PT 현장의 상담 기록과 운동 수행 데이터를 구조화하는 **비의료 웰니스·운동 상담 지원 서비스**입니다. 제품의 전체 방향은 [루트 README §7 안전 및 규제 경계](../../README.md#7-안전-및-규제-경계), MVP의 위험 라우팅 기준은 [Django README](../../lightone_v2_django/README.md#위험-라우팅-기준-auto--review--block)를 기준으로 합니다.

## 1. 서비스가 제공하는 것

- 운동 기록, RPE, 통증 반응, 트레이너 메모의 구조화
- QS/JATC 등 상담 참고 지표의 산출
- AUTO / REVIEW / BLOCK 라우팅을 통한 트레이너 검토 우선순위 표시
- 회원에게 설명 가능한 비의료 상담 보고서 초안 생성
- 촬영 조건과 입력 품질을 확인하는 보조 QC 정보

## 2. 서비스가 제공하지 않는 것

LIGHTONE V2는 다음 기능이나 표현을 제공하지 않습니다.

- 질병, 손상, 자세 이상에 대한 의료 진단
- 통증 원인 확정 또는 질환 위험도 판정
- 치료, 재활 치료, 의학적 처방, 임상 의사결정 지원
- AI 단독 판단에 따른 운동 처방 자동 확정
- 치료 효과, 질병 예방, 임상 개선 효과 주장
- 의료기관 또는 의료인의 판단을 대체한다는 표현

## 3. 표현 가이드

| 금지 표현 | 권장 표현 |
|-----------|-----------|
| 진단 | 기록, 확인, 상담 참고 |
| 치료 | 관리, 컨디셔닝, 운동 상담 |
| 처방 | 운동 제안, 프로그램 조정 제안 |
| 재활 | 컨디셔닝, 운동 상담 플랜 |
| 통증 원인 분석 | 통증 반응 기록 |
| 위험도 판정 | 주의 신호, 트레이너 확인 필요 |
| AI가 판단 | 트레이너가 검토 |

## 4. 라우팅 문구 기준

- `AUTO`: 일반 상담 보고서 생성 가능. 단, 트레이너 최종 확인 후 회원에게 전달합니다.
- `REVIEW`: 통증 반응 또는 품질 경계값이 있어 트레이너 확인이 필요합니다.
- `BLOCK`: 세션 중단 권고 신호입니다. 의료 진단이 아니며, 필요 시 의료 전문가 상담을 안내합니다.

## 5. 화면·보고서 검수 체크

- [ ] “진단”, “치료”, “처방”, “완치”, “질병”, “환자” 등의 의료 표현이 없는가?
- [ ] 통증 관련 문구가 “원인”이 아니라 “반응” 또는 “기록”으로 표현되는가?
- [ ] AI 또는 시스템이 최종 판단자가 아니라 트레이너 검토 보조 도구로 설명되는가?
- [ ] BLOCK 상태에서 의료 전문가 상담 안내가 포함되는가?
- [ ] 외부 제출 자료는 [모두의창업 요약](../submission/modu-startup-summary.md)의 비의료 문구와 일치하는가?

## 6. 변경 관리

새 기능, 랜딩페이지, 보고서 템플릿, 영업자료가 추가될 때는 이 문서를 기준으로 사전 검수합니다. 실행 문서의 문구를 바꾸는 경우 [Windows 실행 가이드](../execution-guide/windows-run-guide.md)와 [기존 RUN_WINDOWS](../../lightone_v2_django/RUN_WINDOWS.md)의 안내 문구도 함께 확인합니다.
