# 모두의창업 2기 제출 요약

이 문서는 LIGHTONE V2를 모두의창업 2기 심사 관점에서 1~2페이지로 설명하기 위한 요약입니다. 상세 전략은 [루트 README §10 모두의창업 2기 대응 전략](../../README.md#10-모두의창업-2기-대응-전략), 구현 근거는 [Django README](../../lightone_v2_django/README.md)를 참조합니다.

## 1. 한 줄 소개

LIGHTONE V2는 PT샵·헬스장·필라테스 센터가 회원의 운동 기록, 통증 반응, 생활습관, 상담 메모를 구조화해 **재등록 상담과 컨디셔닝 프로그램 설계에 활용하는 비의료 PT 상담 보고서 SaaS**입니다.

## 2. 해결하려는 문제

- PT 상담 기록이 카카오톡, 사진첩, 메모장, 인바디 결과지에 흩어져 있습니다.
- 트레이너는 상담 전 자료 정리에 많은 시간을 쓰지만, 회원에게 설명할 구조화된 보고서는 부족합니다.
- 촬영 조건과 기록 방식이 트레이너마다 달라 변화 비교의 신뢰도가 낮습니다.
- AI 체형분석 앱은 많지만, 현장의 재등록 상담과 기록 표준화 문제를 직접 해결하지 못합니다.

## 3. 솔루션

- 운동 기록, RPE, 통증 반응, 트레이너 메모를 한 화면에서 입력합니다.
- QS/JATC 지표로 상담 참고 상태를 요약합니다.
- AUTO / REVIEW / BLOCK 라우팅으로 트레이너 검토 우선순위를 표시합니다.
- 상담 보고서 초안을 생성하되, 최종 판단은 트레이너가 수행합니다.
- V1의 촬영 QC, 조명 정규화, 캘리브레이션 자산은 신뢰도 보조 레이어로 활용합니다.

## 4. 비의료 경계

LIGHTONE V2는 의료 진단, 치료, 처방, 임상 의사결정을 제공하지 않습니다. 통증은 원인 분석이 아니라 “운동 중 반응 기록”으로 다루며, BLOCK은 의료 진단이 아닌 세션 중단 권고 신호입니다. 세부 기준은 [비의료 경계 정책](../governance/non-medical-boundary.md)을 따릅니다.

## 5. 고객과 수익 모델

| 단계 | 고객 | 수익 가설 |
|------|------|-----------|
| 초기 | 1인 트레이너, 소형 PT샵, 필라테스 센터 | 월 구독형 SaaS |
| 중기 | 센터 대표, 교육 운영자 | 셋업비 + SaaS + 교육 과정 |
| 장기 | 기업 복리후생, 전문가 협력 네트워크 | B2B 파일럿, 인증 IP |

가격과 지불의사는 파일럿 인터뷰와 WTP 검증 후 확정합니다.

## 6. 검증 계획

1차 파일럿은 센터 1~3곳, 회원 10~30명, 2~4주 범위로 진행합니다. 주요 지표는 상담 준비 시간, 보고서 생성 성공률, 트레이너 만족도, 회원 이해도, REVIEW/BLOCK 적절성입니다. 자세한 설계는 [파일럿 검증 계획](../validation/pilot-validation-plan.md)을 참조합니다.

## 7. 현재 준비 상태

- 사업 포지셔닝과 비의료 경계 문서화 완료
- Django MVP 기반 더미 데이터 실행 가능
- QS 라우팅과 리포트 화면의 초기 구조 확보
- 개인정보와 파일럿 운영 체크리스트 초안 마련
- Windows 실행 문서 정리 중. 실행 방법은 [Windows 실행 가이드](../execution-guide/windows-run-guide.md)와 [기존 RUN_WINDOWS](../../lightone_v2_django/RUN_WINDOWS.md)를 확인합니다.

## 8. 심사 핵심 메시지

LIGHTONE V2는 “AI가 몸을 진단하는 서비스”가 아니라, PT 현장의 상담 기록과 재등록 문제를 해결하는 비의료 보고서 SaaS입니다. 기술보다 현장 데이터 구조화, 트레이너 검토 흐름, 상담 UX, 교육·인증 IP가 핵심 경쟁력입니다.
