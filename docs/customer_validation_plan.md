# LIGHT ONE 고객 검증 계획

## 1. 대상 인터뷰 계획

| 대상 | 확인할 핵심 주제 | 우선순위 |
|---|---|---:|
| 프리미엄 PT샵 대표 | 재등록 상담, 상담 품질 표준화, 월 구독 지불 조건 | 1 |
| 트레이너 | 상담 준비 시간, 기록 분산 문제, 리포트 검토 부담 | 1 |
| 필라테스 스튜디오 대표 | 회원 변화 설명 자료, 수업 기록 관리, 리포트 도입 장벽 | 2 |
| 교정운동 센터 | 통증 반응 기록, 주의 신호 관리, 컨디셔닝 상담 흐름 | 2 |

## 2. 인터뷰 목표

1. 상담 전 자료 정리에 시간이 드는지 확인한다.
2. 재등록 상담에서 구조화된 리포트가 필요한지 확인한다.
3. 센터장이 트레이너별 상담 품질 표준화에 관심이 있는지 확인한다.
4. AI 단독 판단보다 트레이너 검토 기반 보고서가 더 수용 가능한지 확인한다.
5. 월 구독, 리포트당 과금, 혼합 모델 중 어떤 방식이 초기 도입에 적합한지 확인한다.
6. 파일럿 참여 조건과 동의·비식별화·최소수집 원칙 수용 여부를 확인한다.

## 3. 질문 리스트

### 배경 질문

1. 현재 역할은 무엇인가요? 예: 대표, 센터장, 트레이너
2. 회원 상담은 언제 진행하나요? 예: 최초 등록, 중간 점검, 재등록 전
3. 상담 자료는 현재 어디에 기록하나요?
4. 상담 자료를 다시 찾거나 정리하는 데 불편했던 최근 사례가 있나요?

### 문제 검증 질문

1. 상담 전 회원 기록을 확인하는 데 보통 얼마나 걸리나요? 수치는 [검증필요]로 기록합니다.
2. 자료가 흩어져 상담 준비가 늦어진 최근 사례가 있나요?
3. 재등록 상담에서 회원 변화는 어떻게 설명하나요?
4. 현재 자료만으로 설명하기 어려운 항목은 무엇인가요?
5. 트레이너별 상담 방식 차이가 센터 운영에 영향을 주나요?

### 솔루션 반응 질문

1. 운동 기록, RPE, 통증 반응, 생활습관, 트레이너 메모가 한 장의 보고서로 정리된다면 어디에 가장 먼저 쓰겠나요?
2. 리포트 초안 생성 후 트레이너 확인 단계가 있다면 운영 부담은 어느 정도일까요?
3. AUTO/REVIEW/BLOCK 라우팅은 트레이너 검토 우선순위에 도움이 될까요?
4. 회원에게 “트레이너 검토 완료 리포트”라고 안내하면 신뢰에 도움이 될까요?

### 파일럿 질문

1. 2~4주 소규모 파일럿에 참여할 의향이 있나요? 기간과 범위는 [검증필요]입니다.
2. 동의 기반, 비식별화, 최소수집 원칙에 동의할 수 있나요?
3. 파일럿 담당자와 주간 피드백 시간을 지정할 수 있나요?
4. 파일럿 이후 유료 전환을 검토하려면 어떤 증거가 필요할까요?

## 4. WTP 가격 테스트

> 아래 가격과 패키지는 검증용 가설이며 실제 가격으로 확정하지 않는다. 모든 수치는 [검증필요]이다.

| 패키지 | 대상 | 구성 | 테스트 질문 |
|---|---|---|---|
| Starter | 1인 트레이너 | 회원 기록 정리, 리포트 초안, 트레이너 검토 | 개인 비용으로 월 구독을 검토할 조건은 무엇인가요? |
| Studio | PT샵·필라테스 센터 | 다수 트레이너 계정, 표준 리포트, 센터장 확인 | 센터 운영비로 결제하려면 어떤 성과가 필요하나요? |
| Report Pack | 이벤트성 상담 | 리포트당 과금, 단기 캠페인 활용 | 건당 과금이 월 구독보다 자연스러운 상황은 무엇인가요? |
| Pilot | 파일럿 센터 | 제한 기간, 비식별 피드백, 운영 회고 | 유료 파일럿을 검토하려면 어떤 조건이 필요하나요? |

## 5. 성공 기준

| 기준 | 성공 신호 | 수정 신호 |
|---|---|---|
| 문제 강도 | 최근 사례와 반복성이 확인됨 | “있으면 좋다” 수준에 머묾 |
| 리포트 가치 | 상담·재등록·회원 설명에 사용할 장면이 명확함 | 사용 장면이 모호함 |
| 지불의사 | 조건부 결제 또는 유료 파일럿 논의 가능 | 무료 사용 외 관심 없음 |
| 도입 가능성 | 동의·입력·검토 흐름을 수용 | 운영 부담이 가치보다 큼 |
| 비의료 신뢰 | 트레이너 검토 기반 설명을 선호 | 자동 산출물 책임 우려가 큼 |

## 6. Airtable 저장 증거

Airtable에는 실제 개인정보를 저장하지 않고 비식별 코드와 검증 증거만 저장한다.

| 필드 | 기록 내용 |
|---|---|
| Interview ID | 인터뷰-001 형식 |
| Segment | PT샵 대표 / 트레이너 / 필라테스 / 교정운동 센터 |
| Problem evidence | 문제 발언 요약, 최근 사례 여부 |
| Report reaction | 샘플 리포트 반응 |
| WTP signal | 월 구독 / 리포트당 과금 / 보류 |
| Required proof | 시간 절감, 재등록 상담 활용, 회원 이해도 등 |
| Privacy consent status | 동의 여부와 비식별 처리 여부 |
| Next action | 샘플 발송, 파일럿 제안, 추가 인터뷰 |
