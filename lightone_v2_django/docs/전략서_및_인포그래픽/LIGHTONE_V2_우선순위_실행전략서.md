# LIGHT ONE V2: 가장 현실적인 우선순위 3가지 및 실행 전략서

**작성일:** 2026-07-01  
**대상:** LIGHT ONE 프로젝트 리드 (송광일 님)  
**목적:** 기존 COPD 프로젝트 경험과 LIGHT ONE V2 통합전략서를 바탕으로, 즉시 실행 가능하고 방어력이 높은 우선순위 3가지를 도출하고 구체적인 실행 전략을 수립함.

---

## 1. 현재 상황 진단 및 핵심 결론

기존 COPD 프로젝트는 **데이터 수집 → 정규화 → 모델 예측 → 의사 검토(Human-in-the-loop) → 결과 리포팅**이라는 훌륭한 파이프라인을 가지고 있습니다. 
LIGHT ONE V2 전략서에 따르면, 기존의 'AI 체형분석(측정)'이라는 경쟁이 치열하고 규제 리스크가 높은 포지션에서 벗어나, **"비의료 PT 상담 리포트 SaaS"**로 전환하기로 결정했습니다.

따라서 지금 당장 카메라 캘리브레이션이나 딥러닝 모델 고도화에 시간을 쏟는 것은 위험합니다. 가장 먼저 증명해야 할 것은 **"트레이너가 이 시스템에 데이터를 입력하고, 생성된 리포트를 상담에 활용할 수 있는가?"**입니다. 

이를 바탕으로 도출한 **가장 현실적인 우선순위 3가지**는 다음과 같습니다.

---

## 2. 가장 현실적인 우선순위 3가지 (Top 3 Priorities)

### 🥇 1순위: 핵심 데이터 입력 폼 및 권한 기반 Django 뼈대 구축 (Data Collection Layer)
**왜 중요한가?** 데이터가 들어오지 않으면 QS 점수도, 리포트도 만들 수 없습니다. 기존 COPD의 `accounts` 및 `daily_care` 구조를 활용하여 가장 빠르게 만들 수 있는 부분입니다.
- **실행 목표:** 트레이너가 회원의 세션 기록(운동 종목, RPE, 통증 반응 등)을 입력할 수 있는 화면 완성.
- **세부 작업:**
  - 회원/트레이너 모델 분리 (COPD의 환자/의사 모델 재사용)
  - 전역 로그인 미들웨어 적용 (보안 및 접근 제어)
  - 세션 기록 입력 폼(SessionRecord) 및 더미 데이터 생성 기능 구현

### 🥈 2순위: 규칙 기반 품질 점수(QS) 산출 및 대시보드 시각화 (Logic & Dashboard Layer)
**왜 중요한가?** 복잡한 머신러닝 없이도 즉각적인 가치를 제공해야 합니다. 송광일 님의 강점인 '데이터 정규화 및 통계 시각화' 역량을 가장 잘 보여줄 수 있는 영역입니다.
- **실행 목표:** 입력된 세션 데이터를 바탕으로 QS를 계산하고, AUTO/REVIEW/BLOCK 라우팅 상태를 시각적으로 보여주는 대시보드 완성.
- **세부 작업:**
  - 규칙 기반(Rule-based) QS 엔진 구현 (초기에는 단순 점수 합산 및 가중치 적용)
  - COPD 프로젝트의 `benchmarks` 앱을 참고하여 지표 정규화(0~100%) 로직 적용
  - 트레이너가 한눈에 볼 수 있는 변화 추세 막대/선 그래프 시각화

### 🥉 3순위: 트레이너 검토 기반(HITL) 비의료 상담 리포트 생성 (Reporting Layer)
**왜 중요한가?** 이 프로젝트의 최종 산출물이자 B2B 고객(원장, 트레이너)이 돈을 지불할 핵심 가치입니다. 의료법 위반 소지를 차단하는 방어막 역할도 합니다.
- **실행 목표:** AI(규칙 엔진)가 초안을 만들고, 트레이너가 최종 확인하여 PDF/웹 형태로 출력하는 리포트 기능 완성.
- **세부 작업:**
  - 리포트 초안 생성 로직 구현 (금지 표현 필터링 적용)
  - 트레이너 승인(Approve) 프로세스 추가 (COPD의 의사 검토 플로우 재사용)
  - `pdf_export.js` 등을 활용한 상담용 리포트 출력 기능

## 3. 우선순위 실행을 위한 전략적 접근 (Execution Strategy)

### 전략 1: "버리는 것이 아니라 재배치하는 것이다"
기존에 연구했던 카메라 캘리브레이션, 조명 정규화(Layer 1: Data Quality) 기술을 버리지 마십시오. 다만 지금은 전면에 내세울 때가 아닙니다. 
- **적용:** 이 기술들은 나중에 "생성된 리포트의 신뢰도를 높이기 위한 품질 보증(QC) 레이어"로 포장하여 기술 증빙 자료(GitHub, 부록)로 활용합니다. 

### 전략 2: "AI 예측보다 규칙 기반(Rule-based) MVP가 먼저다"
처음부터 완벽한 AI 모델을 만들려다 보면 시간이 부족합니다. 
- **적용:** 초기 MVP는 하드코딩된 규칙(예: 특정 통증 지수가 3일 연속 높으면 'REVIEW' 판정)으로 작동하게 만듭니다. 데이터가 쌓인 후 XGBoost/LightGBM으로 전환한다는 명확한 로드맵만 제시해도 심사위원/고객을 설득할 수 있습니다.

### 전략 3: "의료적 판단의 완전한 배제"
가장 큰 리스크는 '의료기기'나 '의료행위'로 오인받는 것입니다.
- **적용:** 시스템 내부적으로 철저히 금지어 필터를 적용합니다. (진단 → 측정, 치료 → 컨디셔닝, 처방 → 제안). 그리고 모든 결과물에는 **"본 리포트는 트레이너의 상담을 보조하기 위한 참고 자료이며, 의학적 진단을 대체하지 않습니다."**라는 문구를 강제 삽입합니다.

## 4. 요약 및 다음 단계 제안

광일 님, 지금 가장 필요한 것은 **눈에 보이는 작동하는 웹 서비스(MVP)**입니다. 기존 COPD 프로젝트에서 이미 로그인, 권한 분리, 데이터 정규화, 시각화 경험을 훌륭하게 쌓으셨습니다. 이 코드를 재활용하면 1~2주 안에 LIGHT ONE의 뼈대를 충분히 만들 수 있습니다.

**추천하는 당장의 다음 행동:**
1. **내일 당장:** 로컬 환경에 빈 Django 프로젝트를 생성하고, COPD 프로젝트의 `accounts` 앱과 `middleware.py`를 가져와 회원/트레이너 로그인 구조를 만드세요.
2. **모레:** 회원의 세션 기록을 입력받는 폼(SessionRecord)을 만들고, 더미 데이터를 50개 정도 밀어 넣으세요.
3. **그다음:** 그 더미 데이터를 바탕으로 광일 님의 주특기인 정규화된 그래프 대시보드를 띄우세요.

이 3가지만 돌아가도 사업계획서와 데모 영상의 설득력이 10배는 올라갈 것입니다. 시작할 준비가 되셨다면, Django 초기 세팅 코드나 모델(models.py) 설계 초안부터 바로 작성해 드릴까요?
