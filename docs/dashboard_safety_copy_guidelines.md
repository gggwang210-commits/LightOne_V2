# Dashboard Safety Copy Guidelines

## Forbidden expressions

- AI diagnosis / AI 진단
- pain cause analysis / 통증 원인 분석
- disease risk prediction / 질병 위험 예측
- treatment effect / 치료 효과
- rehabilitation prescription / 재활 처방
- medical-grade analysis / 의료 등급 분석
- AI decides / AI가 결정

## Safe replacement expressions

- AI-assisted consultation support / AI-assisted 상담 지원
- pain response record / 통증 반응 기록
- caution signal / 주의 신호
- conditioning program suggestion / 컨디셔닝 프로그램 제안
- trainer review / 트레이너 검토
- non-medical wellness reference report / 비의료 웰니스 참고 리포트
- Human-in-the-Loop trainer review / 트레이너 검토 기반 확인

## Member-facing copy rules

- 쉬운 단어를 사용하고 불안감을 주지 않습니다.
- “원인”보다 “기록된 변화”를 먼저 설명합니다.
- Good / Check / Review 상태를 사용합니다.
- 의료기관 상담 권고는 red flag caution으로만 표현합니다.

## Trainer-facing copy rules

- REVIEW/BLOCK은 진단이 아니라 운영상 검토 큐입니다.
- 운동 난이도, 동작 범위, 휴식, 회복, 다음 세션 주의사항 중심으로 작성합니다.
- 리포트 발송 전 금지 표현을 검토합니다.

## Admin-facing copy rules

- KPI는 운영 지표로 설명합니다.
- 민감 데이터 접근은 권한과 감사 로그를 전제로 합니다.
- 공개 문서와 샘플 데이터에는 실제 개인정보를 포함하지 않습니다.

## Unsafe vs safe examples

| Unsafe | Safe |
|---|---|
| “AI가 통증 원인을 분석했습니다.” | “입력된 통증 반응 기록을 기준으로 트레이너 검토가 필요한 변화가 감지되었습니다.” |
| “치료 효과가 좋아졌습니다.” | “최근 세션 기록에서 컨디셔닝 수행이 더 안정적으로 기록되었습니다.” |
| “질병 위험이 낮습니다.” | “현재 대시보드에는 의료적 위험 예측 기능이 없으며, 기록 변화만 참고로 표시합니다.” |
| “재활 처방을 생성합니다.” | “다음 세션 전 컨디셔닝 과제 초안을 트레이너가 검토합니다.” |
